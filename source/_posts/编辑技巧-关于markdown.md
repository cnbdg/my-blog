🌟 Markdown 进阶装修指南
1. 漂亮的折叠框（Details）
如果你的文章太长，或者想藏起一些答案/代码，可以用这个：

Markdown
<details>
<summary>点击这里查看隐藏的小秘密 (｡•̀ᴗ-)✧</summary>
哎呀，被你发现了！这里可以放很多内容哦。
</details>
效果：读者点一下文字，内容才会展开。

🎨 针对你的 Fluid 主题的小彩蛋
因为你用的是 Fluid 主题，它其实内置了一些更漂亮的组件（这些叫 Tag Plugins），你可以直接在 .md 文件里用：

各种颜色的提示框：

Markdown
{% note success %}
这是一个成功的提示框！(๑•̀ㅂ•́)و✧
{% endnote %}

{% note danger %}
哎呀，这里出错了！(｡>ㅅ<｡)
{% endnote %}
（除了 success 和 danger，还有 info, warning, primary 等颜色哟！）

行内标签：
如果你想给某个词加个漂亮的底色，可以写：{% label primary @文字内容 %}。

🖼️ 技能二：插入你喜欢的动漫图片
在 Markdown 里插图有三种“姿势”，咱们选最简单的：

方法 A：使用图床链接（最推荐，不占空间）
你可以把动漫图片传到微博图床或 SM.MS 等网站，拿到链接后这样写：
! [图片描述](图片的网址链接)
例如：
! [我的老婆](https://example.com/anime-girl.jpg)

方法 B：存放在本地文件夹（最稳妥）
在你的 source 文件夹下创建一个叫 images 的文件夹。

把你的动漫图片放进去（假设叫 hatsune.jpg）。

在文章里写：
![初音未来](/images/hatsune.jpg)

🎨 进阶：动漫图片还能这么玩！

## 一、 网页视频的“天菜”：.webm 格式
在配置动态壁纸时，为什么优先选择 `.webm`？

* **极高压缩率**：同样的画质，体积比 `.mp4` 更小，网页加载飞快！
* **支持透明度**：这是 `.mp4` 做不到的，可以实现透明背景的动态效果。
* **开源免费**：对开发者极其友好。

**💡 转换小贴士**：
可以使用 FFmpeg 指令一键转换：
`ffmpeg -i input.mp4 output.webm`

## 二、 Hexo `_config.yml` 关键参数复习
这是博客的“大脑”，几个关键项的作用要记牢：

| 参数 | 作用 | 备注 |
| :--- | :--- | :--- |
| `url` | 你的博客网址 | 部署到 GitHub Pages/Cloudflare 必填 |
| `permalink` | 文章链接格式 | 推荐使用 `:title.html` |
| `post_asset_folder` | 是否开启资源文件夹 | 设为 `true` 后，存文章图片更方便 |
| `updated_option` | 更新时间逻辑 | 建议设为 `mtime`（按文件修改时间自动更新） |

## 三、 路径与命名的“捉虫”笔记
背景图片加载不出？通常是以下三个调皮的原因：

1.  **文件名不匹配**：电脑是死板的，`image.png` 和 `image-.png`（多个横杠）完全是两个东西！
2.  **路径深度**：图片必须放在 `public` 文件夹对应的路径下，配置里的路径要与 GitHub 仓库结构严格对齐。
3.  **中文路径过敏**：虽然支持中文，但有时服务器环境会导致中文文件夹路径解析失败，建议优先使用英文命名。

## 四、 Git 填坑连招
当你在多端操作或手动修改 GitHub 文件后，推送报错怎么办？

### 1. 报错：Updates were rejected (远程有新内容)
**方案 A（温柔同步）**：
```bash

git pull origin main  # 先拉下来
# 此时如果进入 Vim 界面（看下方逃生指南）
git push             # 再推上去

. Vim 编辑器“逃生指南”
当你执行 git pull 弹出那个满是波浪线的黑窗口时，不要慌：

按键盘左上角 Esc 键。

输入 :wq（英文状态冒号，保存并退出）。

按 Enter 回车

```
